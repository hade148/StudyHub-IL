## איך להריץ את הסקריפט להזנת הנתונים

### דרישות מוקדמות
1. וודא ש-DATABASE_URL מוגדר בקובץ `.env` בתיקיית `server`
2. וודא שבסיס הנתונים Azure PostgreSQL זמין

### הרצת הסקריפט
```bash
cd server
npm run seed
```

הסקריפט יבצע את הפעולות הבאות:
1. יצירת משתמשים לדוגמה (מנהל וסטודנט)
2. יצירת קורסים עבור כל 21 מוסדות הלימוד (441 קורסים סה"כ = 21 מוסדות × 21 קורסים)
3. יצירת סיכומים לדוגמה
4. יצירת פוסטים בפורום לדוגמה
5. יצירת כלים לדוגמה

### כניסה עם משתמשי הדוגמה
לאחר הרצת הסקריפט, תוכל להתחבר עם:
- **מנהל**: admin@studyhub.local / password123
- **סטודנט**: student@studyhub.local / password123

## שימוש במוסדות וקורסים במערכת

### בממשק המשתמש (Frontend)
המוסדות והקורסים זמינים בממשקים הבאים:
1. **דף הרשמה** (`RegisterPage.tsx`) - בחירת מוסד לימודים
2. **עריכת פרופיל** (`EditProfileModal.tsx`) - בחירת מוסד לימודים
3. **העלאת סיכום** (`UploadPage.tsx`) - בחירת קורס (מוצגים כל הקורסים מכל המוסדות)
4. **פתיחת שאלה בפורום** (`NewQuestionPage.tsx`) - בחירת קורס (מוצגים כל הקורסים מכל המוסדות)
5. **עריכת סיכום** (`EditSummaryDialog.tsx`) - בחירת קורס (מוצגים כל הקורסים מכל המוסדות)
6. **עריכת שאלה בפורום** (`EditForumPostDialog.tsx`) - בחירת קורס (מוצגים כל הקורסים מכל המוסדות)

כל הרכיבים האלה משתמשים ב-API לטעינת הנתונים באופן דינמי.

### API Endpoints
- `GET /api/courses/institutions` - קבלת רשימת כל המוסדות הייחודיים
- `GET /api/courses` - קבלת כל הקורסים מכל המוסדות (עם פירוט המוסד לכל קורס)
- `GET /api/courses?search=<term>` - חיפוש קורסים לפי שם/קוד
- `GET /api/courses?institution=<name>` - סינון קורסים לפי מוסד (אופציונלי)

### עקרונות תצוגה
1. **רשימת מוסדות** - מוצגת רק כאשר המשתמש צריך לבחור מוסד לימודים (הרשמה/עריכת פרופיל)
2. **רשימת קורסים** - מוצגת רק כאשר המשתמש צריך לבחור קורס, כולל שם המוסד בסוגריים
3. **אין סינון בצד לקוח** - כל הקורסים מוצגים ללא סינון לפי מוסד בצד הלקוח
4. **עקביות מלאה** - אותה רשימה מוצגת בכל מקום במערכת

## הוספת מוסדות וקורסים נוספים

### הוספה ידנית דרך ממשק מנהל
מנהלים יכולים להוסיף קורסים חדשים דרך:
```
POST /api/courses
```

### הוספה דרך הסקריפט
ערוך את `server/prisma/seed.js` והוסף פריטים לרשימות `institutions` או `coursesList`.

## תכונות נוספות

### רשימות גלילה
כל תפריטי הבחירה במערכת תומכים בגלילה אוטומטית כאשר יש הרבה אפשרויות.

### טעינה דינמית
המערכת טוענת את המוסדות והקורסים מבסיס הנתונים בזמן אמת, כך שכל שינוי מתעדכן מיד בכל החלקים של האפליקציה.

### טיפול בשגיאות
אם ה-API לא זמין, המערכת חוזרת לרשימה מקיפה של כל המוסדות והקורסים כברירת מחדל.
